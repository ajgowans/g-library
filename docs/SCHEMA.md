# Schema

## comics_master.csv

One row per comic in the collection exports.

Columns:

- `source_tab` - Batman, Other, or Heavy Metal.
- `source_row` - row number in the source CSV.
- `reading_order` - original reading order field when present.
- `comic` - original comic label exactly as entered.
- `title` - parsed title.
- `issue_number` - parsed issue number, if available.
- `story` - story / arc label from the source sheet.
- `collection_status` - Yes/No/etc from the source sheet.
- `year`, `month` - mainly for Heavy Metal.
- `format` - starter format label.
- `notes` - free text.

## facsimiles_batman_seed.csv

One row per Batman-related facsimile or likely facsimile target.

The `owned_matching_issue_in_collection` field means a matching issue number was found in your collection. It does not yet prove whether you own the original, a facsimile, or another reprint.


## facsimiles_batman.csv

One row per facsimile edition target. Facsimiles are treated as first-class collectable items, not just metadata on the original issue.

Key columns:

- `facsimile_id` - Stable internal ID.
- `original_title`, `original_issue` - The issue being reproduced.
- `original_year`, `era` - Useful for sorting and era filters.
- `story_or_significance` - Why the issue matters.
- `original_purchase_strategy` - `Facsimile only`, `Facsimile preferred`, `Original or facsimile acceptable`, or `Original preferred if affordable`.
- `facsimile_status` - Released, announced, repress/watch item, or tracking candidate.
- `facsimile_release_date` - Known or approximate date where available.
- `priority` - High / Medium / Low.
- `andrew_score` - 1-5 relevance score for this collection.
- `want_original_eventually` - Whether an original is realistic/desirable later.
- `original_owned_in_collection` - Script-generated by matching the original issue against `comics_master.csv`.
- `facsimile_owned_confirmed` - Manual field. This should be set by you because the current source CSV does not always distinguish original vs facsimile vs other reprint.
- `ownership_action` and `shopping_status` - Script-generated report fields.
- `source_url` - Plain URL evidence for the facsimile/release where known.

Rule: only manually edit fields that require collector judgement: `facsimile_owned_confirmed`, `priority`, `andrew_score`, notes, and ownership preferences.


## Audit reports

`scripts/audit_collection.py` reads `data/processed/comics_master.csv` and writes review-only reports. It does not mutate collection data.

Generated files:

- `reports/collection_audit.csv` - one row per data-quality or collector-uncertainty finding.
- `reports/ownership_to_confirm.csv` - rows where the physical copy type, cover type, or ownership status needs manual confirmation.
- `reports/duplicate_candidates.csv` - normalized title/issue duplicates to review.
- `reports/collection_audit_summary.csv` - counts by audit check.
- `reports/mia_items.csv` - rows marked `MIA`, meaning believed owned but currently not located.
- `reports/tbc_story_items.csv` - rows whose story/event classification is still `TBC`.
- `reports/format_to_confirm.csv` - owned rows whose physical copy type remains uncertain.

The audit treats uncertain copy types as expected review work, not necessarily bad data. For example, `Original/Reprint Unknown` and `Facsimile/Reprint?` are useful placeholders until the physical comic is checked.

Status convention:

- `Yes` - owned and present.
- `No` - not owned.
- `MIA` - believed owned but currently not found.

Story convention:

- `TBC` - owned or tracked, but story/event/shelf classification still needs research.


## event_checklists.csv

One row per issue in a story/event reading checklist. These are reference rows, not ownership rows.

Columns:

- `event_id` - Stable lowercase event key, such as `zero_hour`.
- `event_name` - Display name.
- `reading_order` - Sortable order within the event.
- `series`, `issue_number` - Issue identity used for matching against `comics_master.csv`.
- `role` - `core`, `prelude`, `tie_in`, or a later event-specific role.
- `priority` - `required`, `gotham`, or `optional`.
- `collecting_status` - `collect`, `review`, or `skip`.
- `source_url` - Checklist source.
- `notes` - Manual context.

`scripts/event_report.py` compares this file with the master collection and writes:

- `reports/event_<event_id>_checklist.csv`
- `reports/event_owned.csv`
- `reports/event_missing.csv`

Current seeded events:

- `zero_hour` - Zero Hour: Crisis in Time.
- `bloodlines` - Bloodlines 1993 annual crossover.
